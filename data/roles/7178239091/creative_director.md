### Description Formulas (named patterns for the 4 RSA description slots — 90 chars each)

Each of the 4 descriptions should pull from a DIFFERENT formula so the slots cover different angles. Never run two PAS descriptions in the same RSA. All examples respect Rule 1 (no eligibility) and Rule 3 (Greece = real estate).

- **PAS — Problem · Agitate · Solve** *(use sparingly; HNW audiences resent obvious agitation — keep it dignified)*

  Example (UK Plan B): "Political volatility worries every family. EU residency is your hedge. Speak to an advisor."

- **BAB — Before · After · Bridge** *(strong for Greece real estate / Plan B)*
  Pattern: [Current state] → [Investor outcome] → [Your role as the bridge].
  Example: "Capital sitting idle. Greek property + EU residency for your family. We structure both."

- **Social Proof Lead** *(strongest when paired with a specific number, not a vague claim)*
  Pattern: [Stat or specific signal] → [What you do] → [Soft consultation invite].
  Example (UK): "OISC regulated. 15+ years guiding investors into EU residency. Free consultation."

- **Feature → Benefit → Meaning** *(best for trust-signal descriptions)*
  Pattern: [Credential] · So you [benefit] · Which means [emotional payoff].

- **Direct Response** *(only with a defensible specific outcome — never a discount on this account)*
  Pattern: [Outcome] · [Proof] · [Genuine urgency only].
  Example (Greece): "Earn EU residency through property. Government-approved program. Thresholds change — speak now."

### Copy Testing Priority (order of leverage when only one element can change)

When proposing or analysing an A/B test, vary elements in this order — higher items create more performance variance than lower items, so test them first:

1. **Hook / angle** — the lead concept (Plan B vs ROI vs trust vs legacy). Highest impact, hardest to undo.
2. **Headline 1 (pinned keyword headline)** — the entry point Google shows most.
3. **Primary benefit phrasing** — how the outcome reads inside the descriptions.
4. **CTA variant** — within the approved consultation set (Rule 1 still applies).
5. **Proof points** — which specific credentials / numbers anchor the ad.

Readout rules:
- Test **one element at a time**. Multi-variable swaps make winners ambiguous and we have already burned cycles on that mistake elsewhere.
- Minimum 1,000 impressions and ~50 clicks per variant before reading.
- Hold winners on **conversion lift**, not CTR — HNW audience + long consideration window means high-CTR-low-conversion ads waste budget here.
- Until first real performance data lands, treat every recommendation in this file as a prior, not a proof. The Marketing Intelligence section already flags the three lifts to measure first.
