You can now follow what's shipping in the app from a dedicated **Changelog** page (top nav, next to Studio).

Each entry is one short note about something that changed — what's new, what's fixed, what's been improved — without the engineering details. Filter by type (fix / feature / improvement / research / breaking) or by tag, and search the body text.

Entries are markdown files in `data/changelog/` so they're versioned in git alongside the code that ships them.
